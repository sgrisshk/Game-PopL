# Итерация 1: определения данных

В этой итерации создан только каркас функционального ядра с определениями данных по стилю HtDP.

Сделано:
- Определён тип `Tile` как один из пяти вариантов: `Wall`, `Floor`, `Key`, `Door`, `Exit`.
- Определена структура `Pos` как `(make-pos x y)` для координат в сетке.
- Определена структура `Player` как `(make-player pos keys)`.
- Определена структура `World` как `(make-world grid player status)`.
- Сетка представлена как список списков тайлов: `[List-of [List-of Tile]]`.
- Для каждого определения добавлены:
  - интерпретация,
  - минимум 2 конкретных примера,
  - шаблон функции-потребителя (template).

Важно:
- Игровая логика (движение, проверки столкновений, победа/поражение и т.д.) не реализовывалась.
- Мутации нет, подготовлен чисто функциональный каркас для следующего шага.
