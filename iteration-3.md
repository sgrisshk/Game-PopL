# Итерация 3: логика движения игрока

В этой итерации добавлена чистая логика перемещения игрока в `core.rkt` по рецепту HtDP.

Добавлено:
- `move-player : World KeyEvent -> World`
- `next-pos : Pos KeyEvent -> Pos`
- `valid-move? : World Pos -> Boolean`
- Вспомогательные функции:
  - `movement-key?`
  - `in-grid?`
  - `tile-at`
  - `nth-item`

Правила движения:
- Обрабатываются только стрелки (`up/down/left/right`).
- В `Wall` ходить нельзя.
- При невалидном ходе мир возвращается без изменений.
- Взаимодействия с ключами/дверями/выходом пока не реализованы.

Тесты:
- Добавлены `check-expect` тесты для `move-player` (>= 3).
- Добавлены тесты для helper-функций (`next-pos`, `valid-move?`, и др.).

Архитектура:
- Всё реализовано функционально, без мутаций и side effects.
